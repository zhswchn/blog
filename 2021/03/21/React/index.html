React.js 一个用于构建用户界面的JavaScript库

React

React 是一个将数据渲染为HTML视图的JavaScript库

  1. 原生JS操作DOM繁琐，效率低
  2. 原生JS直接操作DOM，浏览器会进行大量的重绘重排
  3. 原生JS没有组件化编码方案，代码复用率低

React特点:

  1. 组件化模式，声明式编码，提高开发效率及组件复用率
  2. RN中可以用React开发移动端App
  3. 使用虚拟DOM+Diffing算法，减少DOM的交互

虚拟DOM:

原生JS修改DOM时是直接替换所有的节点，不管有没有发生变化

通过Diffing算法将数据转为虚拟DOM，在渲染为真实DOM
当新增DOM节点数据时，虚拟DOM会对比，相同的已有的节点不会变，只渲染新增的节点

循环需要key值，就是diff用来区分项的，没有key就无法复用DOM节点

去React官网下载React文件，新建html文件，引入React和ReactDOM的js文件以及babel文件(解析JSX)，可以获得全局的React和ReactDOM对象

两种创建虚拟DOM的方式

第一种是用jsx

第二种是用js写

  1. 虚拟DOM本质是Object类型的对象
  2. 虚拟DOM属性较少，真实DOM属性较多，虚拟DOM只在React内部使用
  3. 虚拟DOM最终会被React转为真实DOM

jsx

全称：JavaScript XML

用来简化创建虚拟DOM，不是字符串不是html/xml标签，最终产生一个js对象

  1. 定义虚拟DOM不用””
  2. jsx中js表达式要用{}括起来，只能放表达式，js语句会报错
  3. CSS样式要用className，不能用class
  4. style样式要用 {{ }} 括起来，如:{{color:"red",fontSize:"16px"}}
  5. 只有一个跟标签，标签必须闭合
  6. 标签首字母小写，将转为html中同名标签，没有相对应的会报错
  7. 大写会去渲染相对应的组件，没有会报错
  8. 遇到<以html标签解析，遇到{以js语法解析
  9. 浏览器不能直接解析

表达式: 一个表达式会产生一个值，可以放在任何一个需要值的地方
语句: 控制语句，循环语句

复习类：

  1. 类中的构造器不是必须写的，要对实例进行一些初始化才写

state总结

  • state是组件中重要的对象,必须是个对象
  • 通过更新组件的state来更新对应的页面显示(render()重新渲染)
  • 组件中的render()中的this为组件的实例对象
  • 组件中自定义的方法中的this为undefined，可通过bind()绑定this或者用箭头函数
  • state中的数据不能直接修改或更新，要用setState()

组件标签中写的属性都会保存到该组件的props中，props是只读的，组件内不允许修改，这个也是单向数据流

标签体内容是一个标签属性,可以通过this.props.children获取

类组件中的constructor几乎不用写，如果不初始化state或者不进行方法绑定，则不需要在类里面写构造器

然而state可以不写在里面，方法绑定也可用箭头函数代替，所以不用写

constructor中应在this使用之前调用super(props)，否则使用this.props会导致undefined，注意只有this.props会导致，props完全没问题

组件生命周期

  1. 组件从创建到死亡它会经历一些特定的阶段。
  2. React组件中包含一系列勾子函数(生命周期回调函数), 会在特定的时刻调用。
  3. 我们在定义组件时，会在特定的生命周期回调函数中，做特定的工作。

生命周期的三个阶段（旧）

  • 初始化阶段: 由ReactDOM.render()触发—初次渲染
    • constructor()
    • componentWillMount()
    • *render()
    • *componentDidMount() 做初始化，如网络请求、开启定时器
  • 更新阶段: 由组件内部this.setSate()或父组件重新render触发
    • componentWillReceiveProps 父组件第二次render从这触发
    • shouldComponentUpdate() this.setSate()从这触发 返回一个布尔值，默认为true，为false时不执行下面的生命周期
    • componentWillUpdate() forceUpdate()从这触发
    • *render()
    • componentDidUpdate()
  • 卸载组件: 由ReactDOM.unmountComponentAtNode()触发
    • *componentWillUnmount() 做收尾工作，如关闭定时器

*常用

生命周期的三个阶段（新）

  • 初始化阶段: 由ReactDOM.render()触发—初次渲染
    • constructor()
    • getDerivedStateFromProps
    • render()
    • componentDidMount()
  • 更新阶段: 由组件内部this.setSate()或父组件重新render触发
    • getDerivedStateFromProps
    • shouldComponentUpdate()
    • render()
    • getSnapshotBeforeUpdate
    • componentDidUpdate()
  • 卸载组件: 由ReactDOM.unmountComponentAtNode()触发
    • componentWillUnmount()

即将废弃的勾子

  1. componentWillMount
  2. componentWillReceiveProps
  3. componentWillUpdate

现在使用会出现警告，下一个大版本需要加上UNSAFE_前缀才能使用，以后可能会被彻底废弃，不建议使用。

static getDerivedStateFromProps(props,state)返回一个对象来更新state，如果返回null则不更新任何内容，这个声明周期比较少用，除非state的值在任何时候都取决于props，这时可以使用（不是必须使用，会导致代码冗余，组件难以维护）

getSnapshotBeforeUpdate() 更新之前获取快照，能在发生改变之前从DOM中捕获一些信息，此声明周期的任何返回值都将会作为参数传递给componentDidUpdate(preProps,preState,snapshotValue)，应返回snapshot(快照)值或null，不常用(比上一个有点用)

BrowserRouter & HashRouter

  1. 底层原理不一样：
    1. BrowserRouter使用的是H5的 history API,不兼容IE9及以下版本
    2. HashRouter使用的是URL的哈希值
  2. url表现形式不一样
    1. BrowserRouter的路径中没有#，例如：localhost:3000/demo/test
    2. HashRouter的路径包含#,例如：localhost:3000/#/demo/test
  3. 刷新后对路由state参数的影响
    1. BrowserRouter没有任何影响，因为state保存在 history 对象中。
    2. HashRouter制新后会导致路由state参数的丢失
  4. 备注：HashRouter可以用于解决一些路径错误相关的问题

redux

redux是什么

  1. redux是一个专门用于做状态管理的JS库(不是react插件库)。
  2. 它可以用在react, angular, vue等项目中, 但基本与react配合使用。
  3. 作用: 集中式管理react应用中多个组件共享的状态。

什么情况下需要使用redux

  1. 某个组件的状态，需要让其他组件可以随时拿到（共享）。
  2. 一个组件需要改变另一个组件的状态（通信）。
  3. 总体原则：能不用就不用, 如果不用比较吃力才考虑使用。

三个核心

  • action
  • reducer
  • store

action

动作对象,包含2个属性：

  • type：标识属性，字符串，唯一值，必选
  • data：数据属性，任意值，可选

{ type:'add_type', data:{name: 'str', value:'zs'} }

reducer

  • 初始化状态，加工状态
  • 加工时，依据旧的state和action产生新的state 纯函数

store

  • 将state、action、reducer联系在一起的对象
  • getState()：获取state
  • dispatch(action)：分发action，触发reducer自更新，产出新的state
  • subscribe(listener)：注册监听，产生新的state时，自动调用

redux异步

redux默认不能进行异步处理，可以安装异步中间件来执行异步任务

react-redux

  1. 一个react插件库
  2. 专门用来简化react应用中使用redux

react-Redux将所有组件分成两大类

UI组件

  1. 只负责 UI 的呈现，不带有任何业务逻辑
  2. 通过props接收数据(一般数据和函数)
  3. 不使用任何 Redux 的 API
  4. 一般保存在components文件夹下

容器组件

  1. 负责管理数据和业务逻辑，不负责UI的呈现
  2. 使用 Redux 的 API
  3. 一般保存在containers文件夹下

API:

  1. Provider：让所有组件都可以得到state数据
  1. connect：用于包装 UI 组件生成容器组件
  1. mapStateToProps：将外部的数据（即state对象）转换为UI组件的标签属性
  2. mapDispatchToProps：将分发action的函数转换为UI组件的标签属性

也就是说都可以在UI组件的props中获取
