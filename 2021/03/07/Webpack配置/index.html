Webpack配置

本文最后更新于：2022年7月3日 晚上

记录 Webpack 常用配置及各类插件、技巧

前言

学的时候还是 4.x，现在 webpack5 都有了，学这些配置也一直没用上，学了没用等于没学，记一下大概的流程吧，有机会用到在说吧。

Webpack 配置技巧

  1. webpack.config.js 配置时先写最外层的再配里面的
  • mode
  • entry
  • output
  • module
  • plugins
  • devServer
  1. 一些 API 可能不记得了，可以在配置文件里面使用 ts 类型来做代码提示

配置依赖

webpack 三大件

  • webpack
  • webpack-cli
  • webpack-dev-server

处理 ES6+ 的 babel 核心三大件

  • babel-loader
  • babel-core
  • babel-preset-env

及其他 babel 按需求安装

  • babel-plugin-transform-runtime
  • babel-plugin-transform-decorators
  • babel-plugin-transform-decorators-legacy
  • …

处理 css 样式， loader 为倒序，一层层处理

sass => css => style

  • style-loader
  • css-loader
  • sass-loader
  • node-sass

处理 html

webpack5 如果要自定义动态定义 title 得用 index.ejs

  • html-webpack-plugin

打包前清空文件

webpack5 output 自带，不需要 clean-webpack-plugin

webpack5 内置了资源处理模块，file-loader 和 url-loader 都可以不用安装
