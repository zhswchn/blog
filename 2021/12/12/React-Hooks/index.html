React Hooks

转 React 技术栈了，最近复习一下 Hooks 相关知识在此记录下来，主要时一些常用的 Hook 。

Hooks 是 React 16.8 新增的特性，它可以让你在不编写 class(类组件) 的情况下使用 state 以及其他的 React 特性。

useState

useStat用于在函数组件中添加 state 的 Hook。

可以设置一些复杂数据类型，数组对象等，但是 setCount 在设置变量的时候是整个替换，而不是合并数据，要注意手动合并，并且在更新数据时会重新渲染组件。

useEffect

useEffect用于在函数组件中执行一些副作用操作。

副作用：

React 把在更新 DOM 后运行的一些额外带代码称之为副作用，比如发送网络请求、手动更改 DOM、设置/销毁定时器等等。其中一些在操作之后可以忽略掉，比如发送网络请求，另一些需要手动清除，比如定时器。

useEffect 做了什么？ 通过使用这个 Hook，你可以告诉 React 组件需要在渲染后执行某些操作。React 会保存你传递的函数（我们将它称之为 “effect”），并且在执行 DOM 更新之后调用它。

每次 DOM 渲染完都会执行一次 useEffect的回调，包括更新 DOM 后。

但是每次重新渲染都会生成新的Effect，替换之前的，也就是说每次传递给 useEffect 的函数都是一个新的函数。

可以设置依赖项，根据依赖项控制 useEffect执行，这样可以减少不必要的计算，从而优化性能。

如果依赖项为[]空数组，那么只会在第一次渲染时执行一次，后续 DOM 更新渲染后不会执行。

useEffect 每个 effect 都可以返回一个清除函数，这个函数会在组件卸载的时候清除副作用，比如清除定时器。

简单点形容：

useReducer

useState 的替代方案。它接收一个形如 (state, action) => newState 的 reducer，并返回当前的 state 以及与其配套的 dispatch 方法。

在某些场景下，useReducer 会比 useState 更适用，例如 state 逻辑较复杂且包含多个子值，或者下一个 state 依赖于之前的 state 等

useContext

可以实现从上到下跨多层组件传递数据，兄弟组件数据共享，或者配合 useReducer 组合 redux

基本使用

实现数据共享

共享中心

把要共享数据的组件包裹起来

组件使用通信

有点像我写Vue拆分组件时，数据和方法定义在父组件中，传递给子组件使用，子组件通过父组件获取和更新数据，主要起到一个数据中心的作用。

同样的道理，把共享中心的useState换成useReducer即可组成一个简单的Redux

useRef

useRef 返回一个可变的 ref 对象，其 .current 属性被初始化为传入的参数（initialValue）。返回的 ref 对象在组件的整个生命周期内保持不变。

Ref Vue中也有，一般用于获取子组件实例，React 中也可以用来访问 DOM。

还可以用来保存任何可变变量

useCallback

一般用来在函数组件中缓存计算函数，避免重复的无用计算

useCallback会把callback返回，并将结果 memoizedCallback 缓存，只有当依赖[deps]更新时才会返回新的函数。

useMemo

这个用来返回计算结果，也是在依赖改变时重新计算，有助于减少每次渲染时的计算开销，它会在渲染期间执行。

它的第一个参数也是回调函数，因为是缓存结果，所以必须要return一个返回值

useImperativeHandle

正常情况下 ref 是不能挂在到函数组件上的，因为函数组件没有实例。

useImperativeHandle 可以让你在使用 ref 时自定义暴露给父组件的实例值。

useImperativeHandle 应当与 forwardRef一起使用

forwardRef可以用来获取子组件的 ref

如果要父组件执行子组件中的方法需要useImperativeHandle

createHandle：一个函数，返回一个对象

总结

目前就这些 Hook 较为常用，其他常用的大多为第三方提供的，比如阿里的aHooks。

React 默认会有多余的 render 组件更新哪怕一些方法没有变化却依旧会渲染，这部分的渲染是浪费的，所以部分 Hook 有提供依赖来控制渲染。
