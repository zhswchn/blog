Gulp

本文最后更新于：2022年7月3日 晚上

本篇记录 gulp 的安装和使用方法， gulp 是自动化构建工具，不同于 webpack，
webpack 是基于 js 文件的构建工具, 而 gulp 是基于流的构建工具。

简介

gulp 是什么?

  1. gulp 是一个开源的 JavaScript 开源自动化工具
  2. gulp 是基于 NodeJS 和 NPM 应用的构建工具
  3. gulp 主要用于处理耗时及重复的任务
  4. gulp 可以解决上百种的任务例如：压缩代码合并代码等

gulp 能够压缩代码，合并代码，压缩图片， Sass 转换等等

gulp 提供了很多插件

gulp 是基于流文件的构建工具，就像是工厂里的流水线一样，一层层处理。

常用 API

  • gulp.task(任务名称, 任务处理函数) – 创建任务
  • gulp.src(路径信息) – 找到需要执行任务的文件
    • gulp.src('./src/a.html') – 找到src目录下指定文件
    • gulp.src('./src/*.html') – 找到src目录下指定后缀文件
    • gulp.src('./src/**') – 找到src目录下所有文件
    • gulp.src('./src/**/*') – 找到src目录下所有子目录里面的所有文件
    • gulp.src('./src/**/*.html') – 找到src目录下所有子目录里指定的后缀文件
  • gulp.dest(路径信息) – 执行任务后文件的去处
  • gulp.watch(路径信息, 任务名称) – 监听文件是否发生变化，一旦变化就重新执行对应的任务
  • gulp.series(任务1, 任务2, 任务3, ...) – 按顺序执行多个任务
  • gulp.parallel(任务1, 任务2, 任务3, ...) – 并行执行多个任务
  • gulp.pipe(任务) – 管道函数 接收左侧数据并使用任务处理然后输入到右侧
    • gulp.src('./index.js').pipe(压缩).pipe(混淆).pipe(gulp.dest('dist'))

有两种方式解决：

打包 js

目前 gulp-uglify 可以直接打包压缩 ES6 语法，如果有转 ES5 的需求
可用 gulp-babel，它有两个版本:
